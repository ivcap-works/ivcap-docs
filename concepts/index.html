    How IVCAP Works¶

    IVCAP is a managed execution platform built around four core ideas:

    1. Everything is a service. Any analysis, model, or workflow can be registered as a containerised service with typed inputs and outputs.
    2. Every execution is recorded. Jobs, artifacts, and metadata are all append-only records — giving you a complete, immutable audit trail.
    3. Agents are first-class citizens. Services can autonomously call other services, call LLMs, and orchestrate complex workflows — without any pre-authorisation.
    4. Data and metadata are unified. Artifacts (data blobs) and Aspects (typed metadata) live together in the Data Fabric, making every result queryable, composable, and reproducible.

    The core entities¶

    mermaid graph LR subgraph DF["Data Fabric"] S([Service]) J([Job]) A([Artifact]) Asp([Aspect]) S -->|submits| J J -->|produces| A S -.->|every event\nrecorded as| Asp J -.->|every event\nrecorded as| Asp A -.->|every event\nrecorded as| Asp end

    Entity What it is URN pattern
    Service A registered analytic capability urn:ivcap:service:<uuid>
    Job A single execution of a service urn:ivcap:job:<uuid>
    Artifact Any data blob (image, CSV, model, JSON, …) urn:ivcap:artifact:<uuid>
    Aspect A typed, append-only metadata record on any entity urn:ivcap:aspect:<uuid>
