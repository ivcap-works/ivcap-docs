# Orders (Jobs) — Redirected

!!! info "This page has moved"
    Jobs (formerly called *orders* in older API paths and CLI commands) are now
    documented as part of the **Services & Jobs** reference.

    → **[Services & Jobs API](services.md#jobs)**

The `/1/orders` path remains available in the REST API for backward
compatibility, but new integrations should use `/1/services/{id}/jobs`.
The canonical term throughout the documentation is **job**.
